# Evaluating LLM-Generated Synthetic Datasets for SMS Smishing Detection

Short Message Service (SMS) phishing, commonly referred to as smishing, remains a significant cybersecurity threat because attackers can exploit user trust, mobile device ubiquity, and the perceived immediacy of text messaging. Machine learning methods have shown promise for identifying malicious SMS messages, but model performance depends heavily on the characteristics and quality of the training data. Recent advances in large language models (LLMs) have made it easier to generate synthetic cybersecurity datasets; however, it is not yet clear whether models trained on LLM-generated SMS data perform differently from models trained on human-labeled SMS data.

This project compares the performance of machine learning classifiers trained on two types of SMS datasets: a human-labeled dataset and an LLM-generated dataset. Multiple classification models will be trained and evaluated using accuracy, precision, recall, and F1-score. The study will compare models trained on manually curated data, synthetic data, and a combination of both datasets to determine whether training data source is associated with measurable differences in smishing detection performance when evaluated against real-world SMS messages. The analysis will treat the comparison as a two-tailed test, evaluating whether the datasets produce statistically different model outcomes rather than assuming that one dataset should outperform the other.

To demonstrate practical application, a prototype SMS analysis tool will be developed that classifies user-provided SMS messages and presents an explanation of the features contributing to the prediction. The project’s results will help clarify whether LLM-generated SMS datasets can effectively support cybersecurity classification tasks and whether differences in dataset characteristics influence smishing detection performance when evaluated on real-world SMS messages.
